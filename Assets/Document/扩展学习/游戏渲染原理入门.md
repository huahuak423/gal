# 游戏引擎渲染原理：从数据到画面

## 一、先理清概念：一个比喻

想象你是一个**画家**，要画一幅肖像画：

```
┌─────────────────────────────────────────────────────────────────┐
│                        画室比喻                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   🎨 UV贴图 = "贴纸定位网格"                                     │
│      脸上画好网格线，告诉你"哪块皮肤对应贴纸的哪个位置"            │
│                                                                 │
│   📝 Shader/着色器 = "调色配方书"                                │
│      规定：皮肤怎么调色、光影怎么打、毛孔要不要画                 │
│                                                                 │
│   🧪 材质 = "一罐调好的颜料"                                     │
│      把配方书(shader)里的参数填好，配出具体的颜料                 │
│                                                                 │
│   🔀 渲染管线 = "整个绘画流水线"                                  │
│      从素描→上色→光影→装框的完整流程                             │
│                                                                 │
│   🕸️ 材质图 = "可视化配方编辑器"                                 │
│      用拖拽节点的方式编写配方书，不用手写代码                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、从3D模型到2D画面：完整旅程

### 第一步：物体数据（食材）

```
一个角色的原始数据：
┌──────────────────────────────────────┐
│  3D模型 (Mesh)                       │
│  - 顶点位置：坐标数组                │
│  - 三角形面：哪三个点组成一个面        │
│  - UV坐标：每个顶点对应贴图的哪个位置  │
│  - 法线：这个面朝向哪里（用于光照）    │
├──────────────────────────────────────┤
│  贴图 (Texture)                      │
│  - 颜色贴图：皮肤是什么颜色           │
│  - 法线贴图：表面凹凸细节             │
│  - 粗糙度贴图：哪里光滑哪里粗糙        │
└──────────────────────────────────────┘
```

**UV贴图详解**：
```
        3D模型                              2D贴图
       ╱╲                                  ┌─────────┐
      ╱  ╲                                 │  头部   │
     ╱ 脸 ╲        UV映射                  │ ┌───┐   │
    ╱      ╲   ─────────────►              │ │脸 │   │
   ╱________╲                               │ └───┘   │
                                           │  身体   │
   模型表面的每个点                         └─────────┘
   都对应贴图上的一个位置
   U = 横坐标 (0~1)
   V = 纵坐标 (0~1)
```

### 第二步：渲染管线（流水线）

渲染管线就是从3D数据到2D像素的**工业流水线**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                        渲染管线流程                                  │
└─────────────────────────────────────────────────────────────────────┘

原始数据                GPU处理                      最终结果
    │                      │                           │
    ▼                      ▼                           ▼
┌────────┐           ┌──────────┐              ┌───────────┐
│ 3D模型 │ ──────►   │ 顶点着色器 │ ──────►     │           │
│ + 材质  │           │ (处理顶点) │             │           │
└────────┘           └──────────┘              │           │
                          │                     │           │
                          ▼                     │  屏幕上的  │
                    ┌──────────┐                │   2D图像  │
                    │  光栅化   │ ──────►        │           │
                    │(三角→像素)│               │           │
                    └──────────┘              │           │
                          │                     │           │
                          ▼                     │           │
                    ┌──────────┐                │           │
                    │ 片段着色器 │ ──────►       │           │
                    │ (计算颜色) │              │           │
                    └──────────┘              └───────────┘
                          │
                          ▼
                    ┌──────────┐
                    │ 后处理   │
                    │(模糊/色调)│
                    └──────────┘
```

#### 各阶段详解：

**1. 顶点着色器 (Vertex Shader)**
```
输入：3D空间中的顶点坐标
输出：屏幕上的2D坐标

工作：把3D坐标"投影"到2D屏幕上

比喻：把立体雕塑的照片拍下来，确定每个点在照片的哪个位置
```

**2. 光栅化 (Rasterization)**
```
输入：三角形顶点
输出：被三角形覆盖的所有像素

工作：把"几何图形"变成"像素点阵"

比喻：素描轮廓 → 填充像素格子
```

**3. 片段着色器 (Fragment Shader)**
```
输入：每个像素的位置、UV坐标、光照信息
输出：这个像素应该是什么颜色

工作：根据材质参数、贴图、光照计算最终颜色

比喻：给每个像素点上色，考虑光线、材质、纹理
```

## 三、Shader（着色器）是什么？

**Shader = 运行在GPU上的小程序**

```
┌─────────────────────────────────────────────────────────────┐
│                    Shader 的本质                             │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   CPU（处理器）              GPU（显卡）                     │
│   ┌─────────┐              ┌─────────────────┐             │
│   │运行游戏逻辑│              │运行 Shader      │             │
│   │AI、物理...│              │顶点/片段着色器   │             │
│   │串行执行   │              │并行执行（超快）  │             │
│   └─────────┘              └─────────────────┘             │
│                                                             │
│   Shader 就是你写给 GPU 的"染色配方"                         │
│   告诉 GPU 每个像素该怎么上色                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 一个简单的片段着色器示例：

```hlsl
// 伪代码，展示原理
float4 FragmentShader(float2 uv, float3 normal)
{
    // 1. 从颜色贴图采样
    float4 baseColor = texture2D(_MainTex, uv);

    // 2. 计算光照
    float light = dot(normal, _LightDir);  // 点乘计算光照强度

    // 3. 混合颜色和光照
    float4 finalColor = baseColor * light;

    return finalColor;
}
```

这段代码会**并行执行**在屏幕上数百万个像素上，这就是 GPU 快的原因。

## 四、材质 = Shader + 参数

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│   Shader (配方书)                                           │
│   ┌─────────────────────────────────────┐                  │
│   │ property Color _MainColor           │  ← 定义可调参数   │
│   │ property Texture _MainTex           │                  │
│   │                                     │                  │
│   │ float4 color = _MainColor * light;  │  ← 计算逻辑      │
│   └─────────────────────────────────────┘                  │
│                         +                                   │
│   参数值                                                    │
│   ┌─────────────────────────────────────┐                  │
│   │ _MainColor = (1, 0.8, 0.6) 肤色     │                  │
│   │ _MainTex = "skin_texture.png"       │                  │
│   └─────────────────────────────────────┘                  │
│                         =                                   │
│   材质 (Material)                                           │
│   ┌─────────────────────────────────────┐                  │
│   │   一个可以直接用在模型上的"颜料罐"   │                  │
│   └─────────────────────────────────────┘                  │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

**举例**：
- 同一个 "Standard Shader"
- 材质A：红色 + 木纹贴图 = 红木材质
- 材质B：银色 + 金属贴图 = 银色金属材质

## 五、材质图 = 可视化 Shader 编辑器

**不用写代码，用连线的方式写 Shader**

```
传统方式（写代码）：
┌────────────────────────────────┐
│ float4 color = tex2D(_MainTex, uv); │
│ color *= _TintColor;           │
│ return color * light;          │
└────────────────────────────────┘

材质图方式（连线）：
┌────────────────────────────────────────────────────────┐
│                                                        │
│  [贴图采样]──────►[乘法]──────►[乘法]──────►[输出]     │
│                      ▲              ▲                  │
│                      │              │                  │
│               [颜色参数]      [光照强度]               │
│                                                        │
└────────────────────────────────────────────────────────┘

Unity 的 Shader Graph、UE 的 Material Editor 都是这种
```

## 七、总结对比表

| 概念 | 是什么 | 比喻 | 谁创建 |
|------|--------|------|--------|
| **渲染管线** | 从3D到2D的完整流程 | 整个绘画流水线 | Unity/引擎提供 |
| **Shader/着色器** | GPU上的计算程序 | 调色配方书 | 程序员写代码 |
| **材质图** | 可视化shader编辑器 | 拖拽式配方编辑器 | 美术用节点连线 |
| **材质** | Shader + 具体参数 | 调好的一罐颜料 | 美术配置参数 |
| **UV贴图** | 3D表面到2D纹理的映射 | 贴纸定位网格 | 模型师展开UV |

## 八、实际例子：Unity 中的关系

```
1. Unity 提供 "Universal Render Pipeline"（渲染管线）

2. 程序员用 Shader Graph 创建一个 "水面shader"
   ┌─────────────────────────────────────┐
   │ 节点：波纹 + 反射 + 折射 + 高光      │
   └─────────────────────────────────────┘

3. 美术基于这个shader创建两个材质：
   - 材质A：蓝色、大波浪 → 大海
   - 材质B：绿色、小波纹 → 池塘

4. 模型师给水面模型展开UV，绘制水纹贴图

5. 最终：材质拖到模型上 → 渲染管线处理 → 画面上出现水面
```

## 九、扩展：渲染管线的演变

### 传统管线（Built-in Render Pipeline）
- Unity 早期版本使用
- 固定流程，难以自定义

### 可编程渲染管线（SRP）
- **URP (Universal Render Pipeline)**：通用、跨平台、性能好
- **HDRP (High Definition Render Pipeline)**：高端画质、仅支持PC/主机

### 核心区别：
```
Built-in:  引擎控制一切 → 灵活性低
SRP:       你可以控制渲染流程 → 可以自定义写渲染管线
```
